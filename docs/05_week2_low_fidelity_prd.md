# 第2周低保真原型 PRD

## 目标用户

准备开店但缺少选址经验的个人创业者、小店主和加盟候选人。典型用户已经有一个或几个候选位置，希望快速判断该位置适不适合某个品类，并得到可截图转发的判断依据。

## 核心目标

第2周目标不是追求视觉精细度，而是把 MVP 关键页面和用户路径固定下来：

1. 用户能从输入页自然开始，不需要阅读说明。
2. 用户等待分析时知道系统正在处理什么。
3. 报告页能让非技术用户看懂“为什么推荐或不推荐”。
4. 用户能回到输入页修改参数再分析。
5. 历史记录可先用本地存储模拟，暂不做登录。

## 核心场景

用户准备在某个商圈开一家门店：

1. 输入城市、区域/商圈/街道、详细位置。
2. 搜索详细位置后，在地图上确认精确点。
3. 输入或选择开店品类。
4. 可选填写月租金、月预算。
5. 点击生成报告。
6. 等待系统分析周边商圈、竞品、交通和住宅。
7. 查看评分、风险和建议。
8. 修改参数后再次分析，或保存到历史记录。

## 输入字段

| 字段 | 必填 | 说明 |
|---|---|---|
| 城市 | 是 | 用于减少地址解析歧义，例如广州、深圳、上海 |
| 区域/商圈/街道 | 是 | 用于定位大致范围，例如天河区体育西路、南山区科技园 |
| 详细位置 | 否 | 用于搜索候选点和报告备注，最终位置以地图选点坐标为准 |
| 地图选点 | 是 | 用户通过搜索、拖拽、缩放、点击确定经纬度 |
| 开店品类 | 是 | 自由输入，提供常见品类候选 |
| 分析半径 | 是 | 默认 1000 米，可选 500、1000、1500、3000 米 |
| 月租金 | 否 | 用于成本压力估算 |
| 月预算 | 否 | 用于资金压力估算 |

## 输出报告结构

| 模块 | 内容 |
|---|---|
| 一句话结论 | 推荐、谨慎推荐、高风险、不建议 |
| 综合评分 | 0-100 分，可截图直观看懂 |
| 分项指标 | 客流潜力、竞争压力、消费匹配、成本压力、交通可达 |
| 地图周边摘要 | 竞品、住宅、办公、交通、商业配套数量 |
| 风险提示 | 竞争密度、租金压力、客流不稳定、数据置信度 |
| 下一步建议 | 实地观察、竞品访查、租金核实、试营业验证 |
| 分析位置 | 城市、地址备注、经纬度、分析半径 |

## 页面范围

| 页面 | 必须包含 | 验收标准 |
|---|---|---|
| 首页/输入页 | 地址输入、地图选点、品类选择、预算/租金可选输入 | 用户不用看说明也知道怎么开始 |
| 加载页 | 正在分析周边商圈、竞品、交通和住宅 | 等待过程不空白，失败时有重试提示 |
| 报告页 | 评分、分项指标、地图周边摘要、风险、建议 | 报告能截图发给别人看懂 |
| 历史页 | 最近分析记录 | MVP 早期先本地保存，不做登录 |

## 第2周验收

- 有简短 PRD。
- 有页面流程图。
- 有可点击 HTML 原型。
- 原型至少能点击完成：输入页 -> 加载页 -> 报告页 -> 修改参数再分析。
- 原型包含失败重试状态。
- 原型包含本地历史记录入口。
